Siempre usar sangrías para diferenciar elementos html anidados

Siempre cerrar elementos de HTML, aún cuando sean elementos vacíos, ejemplo:
	<h1>Siempre cerrar</h1>
	<br />  <=Siempre serrar aunque sea vacío

Siempre al abrir <html> se tiene que incluir el atributo "lang" para definir el idioma, esto sirve para que los buscadores puedan definir el lenguaje de la página
	es decir:
		<html lang="es">

Se puede (y se debe) usar el atributo "title" en los <p> para decir de que es de lo que se está hablando

Esta permitido definir el width y el height de las imagenes dentro de la misma etiqueta, claro, sin el uso de style

El atributo alt se usa cuando no es posible mostrar el elemento en pantalla, así que en su lugar, muestra el texto,
esto serviría por ejemplo para personas con capacidades diferentes, entonces, en su lugar, en lugar de ver la imágen, la podría "escuchar" con un lector de páginas

El siguiente es HTML Headings

No usar headings sólo para hacer el texto mas grande o en negritas, usarlo para cosas que si sean títulos o los temas que se quieren dar en las páginas web

El tag "<pre>" funciona para que se puedan detectar los espacios en blanco y salto de línea insertados directamente en el código.

<q> sirve para señalar citas cortas, esto hace que ponga el texto contenido entre comillas

<blockquote> sirve para señalar citas largas, esto hace que ponga el sangria

<abbr> para abreviaciones, el significado de la abreviacion se tendrá que poner dentro del atributo tittle, es decir <abbr title="significado de la aberviación">SDLA</abbr>

<address cite=""> contendrá la información de algun contacto del documento o del articulo, lo pone en italicas

<cite> define el titulo de algún trabajo, y lo cambia a italica

<bdo> cambia la dirección en que se imprimen las palabras, ejemplo, en ves de escribir hola con <bdo dir="rtl">hola<bdo> imprimiria aloh
computercode: Existen los siguientes elementos para representar código en html, taless como:

<kbd> Define entrada de teclado

<samp> Define salida de computadora

<code> Define código de programación

<var> Define variable matemática.



<style> siempre debe de ir dentro de <head> si se estan aplicando estilos internos

Diferentes estados de los links para cambiar sus estilos:

a:link
a:visited
a:hover
a:active

El atributo "target" se ocupa así:
<a href=" ... " target="_blank">
existen diferentes resultados para target:

_blank	Abre el documento linkeado en una nueva ventana o pestaña
_self	Abre el documento linkeado en el mismo logar donde fue clickeado
_parent	En el frame padre
_top	En todo el tamaño de la página

bookmarks
Se usan para ir a una parte específica de la página

Cuando se inserta una imagen como link, se tiene que especificar siempre un borde:0 para prevenir un borde negro en IE9

elemento con el id deseado
<a href="#irAlid">

Se pueden añadir bookmarks desde una página diferente, pero lo cargará en la parte deseada:

<a href="html_tips.htm#tips">

Image Maps

usando <map> hace que una sola imágen tenga áreas clickeables

ejemplo

<img src="planets.gif" alt="Planets" usemap="#planetmap" style="width:145px;height:126px;">

<map name="planetmap">
  <area shape="rect" coords="0,0,82,126" alt="Sun" href="sun.htm">
  <area shape="circle" coords="90,58,3" alt="Mercury" href="mercur.htm">
  <area shape="circle" coords="124,58,8" alt="Venus" href="venus.htm">
</map>

En imágenes, siempre, pero siempre se tiene que usar el atributo alt
